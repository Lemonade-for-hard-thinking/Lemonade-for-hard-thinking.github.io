2、使用Wireshark抓包，分析上述Web网站在http改造为https之后的区别

  • 对SSL协议进行分析

​ 由于我的虚拟机的网卡使用是NAT模式，在 NAT 模式下，虚拟机的所有流量实际上会经过主机的网络接口，但由于 NAT 转换的存在，主机上直接运行的 Wireshark 可能看不到原始虚拟机的流量，或看到的流量是经过转换后的主机 IP信息。所以我们在虚拟机内部直接使用Wireshark进行对本地环回接口Loopback的流量进行抓包，因为我们访问的就是 http://localhost:8080/WebGoat或者https://localhost/WebGoat，也就是本地回环地址 (127.0.0.1)。这样的请求流量是在回环接口上进行的，而不是通过物理网络接口传输，而Loopback 是处理所有本地回环流量的接口（即 localhost 或 127.0.0.1 的所有请求）

​ 我们先访问一下http://localhost:8080/WebGoat网址并输入用户名和密码进行登录操作，并抓取一下流量。为了后续的分析，我们抓取完流量之后需要保存，抓包的部分结果如图所示：

​ 我们再次访问一下https://localhost/WebGoat网址并输入用户名和密码进行登录操作，并抓取一下流量。抓取完成之后我们同样需要保存。抓包的部分结果如图所示：

​ 之后我们对于抓取到的流量进行详细分析，我们从多个方面分析：

数据传输的明文与加密：

  • HTTP

    登录时的 POST 请求内容是明文传输的，我们可以在数据包中直接看到用户名和密码。所以任何人在同一个网络中拦截到这些数据包，都可以轻易地获得登录信息。

    在Wireshark中我们使用http.request.method == "POST"可以过滤得到包含用户名和密码的请求，我们查看报文的内容可以发现用户名和密码均是明文传输的，报文中结果如下图，图中我们可以直接看到用户名和密码。

  • HTTPS

    HTTPS 使用 TLS（传输层安全协议）加密所有通信内容，包括用户名、密码等敏感数据。抓包时，我们会发现登录请求的实际内容部分是无法读取的（加密），只能看到一些握手和加密过程的信息。在这种情况下，即使攻击者拦截了数据包，内容也是加密的，无法直接获得登录信息，这大大提高了信息的安全性。

握手过程：

  • HTTPS

    ​ 使用 TLS 来保证传输的安全性，因此在 HTTPS 流量中，我们会看到一系列的握手数据包，例如 Client Hello、Server Hello 等。握手过程中会协商加密算法、交换密钥以确保后续通信是加密的。在Client Hello 或 Server Hello 的数据包中，我们可以看到 SSL 握手过程的信息，包括加密协议、证书信息等。

    ​ 如在下图中，我们使用tls过滤器过滤之后可以看到许多握手报文：

  • HTTP

    而 HTTP则没有这种握手过程，所有请求和响应均直接以明文形式传输。

抓包结果总结

  • HTTP 流量的风险

    ​ HTTP 的数据传输是不安全的，所有内容都是明文。任何截获到这些数据包的攻击者都能看到其中包含的敏感信息（例如用户名和密码）。这使得 HTTP 的登录信息很容易被中间人攻击截获，存在重大安全风险。

  • HTTPS 提供的安全性

    ​ HTTPS 通过加密来保证数据在传输过程中的安全性。在 Wireshark 中抓取到的数据包中，所有敏感数据均已加密，无法直接读取到明文信息。通过 SSL/TLS 握手，HTTPS 使得攻击者即使拦截了数据包，也无法轻易破解数据内容。

对SSL协议的分析

  • SSL/TLS 握手过程

    在 HTTPS 抓包中，你可以看到 Client Hello 和 Server Hello 包，这些数据包用于协商安全加密算法和会话密钥。

    如下图是我们分别打开一个Client Hello 和 Server Hello报文，我们查看报文的Transport Layer Security 部分：

​ 从报文中我们可以看出Client Hello是 TLS 握手的第一个步骤，客户端通过这个消息告诉服务器它支持的各种加密参数，并尝试协商一个共同支持的加密方式，Server Hello 是服务器对客户端的 Client Hello 的回应，用于通知客户端将要使用的协议版本、加密套件以及相关的参数。

​ 实验总结：在抓包分析实验中，我们通过 Wireshark 对比了 HTTP 和 HTTPS 网站在传输过程中数据的不同。首先，使用 HTTP 访问网站时，所有数据，包括用户名和密码，均为明文传输，容易被攻击者窃取。相反，使用 HTTPS 后，数据被加密传输，即便数据包被拦截，攻击者也无法读取其中的敏感信息，此外，我们还分析了 HTTPS 流量中的 SSL/TLS 握手过程，了解了如何通过 Client Hello 和 Server Hello 协议进行加密算法和密钥的协商。通过此次实验，我深刻理解了 HTTP 与 HTTPS 在安全性方面的差异，特别是加密通信如何有效保护数据隐私，防止中间人攻击。
